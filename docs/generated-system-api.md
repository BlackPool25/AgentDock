# Generated System API Reference

Every system generated by AgentDock exposes a single public surface: the **orchestrator** on port 4000. All agent endpoints are proxied through it and gated by each agent's `expose[]` config.

---

## Authentication

All `/api/*` endpoints require a JWT. Get one by logging in first.

### POST /auth/login

Get a JWT token. No authentication required.

**Password:** Set `API_PASSWORD` in your `.env`. If not set, defaults to `JWT_SECRET`.

```bash
curl -X POST http://localhost:4000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"password": "your-api-password"}'
```

**Response:**
```json
{
  "token": "eyJhbGciOiJIUzI1NiJ9...",
  "expiresIn": "24h"
}
```

Use the token in subsequent requests:
```bash
curl http://localhost:4000/api/system/status \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiJ9..."
```

## System Endpoints

### GET /health

System health check. Public — no JWT required.

```bash
curl http://localhost:4000/health
```

```json
{
  "status": "ok",
  "systemId": "abc123",
  "agents": [
    { "id": "my-agent", "status": "running" }
  ]
}
```

`status` is `"ok"` when all agents are running, `"degraded"` otherwise.

### GET /api/system/status

Detailed status of all agents. Requires JWT.

```bash
curl http://localhost:4000/api/system/status \
  -H "Authorization: Bearer {token}"
```

```json
{
  "systemId": "abc123",
  "status": "running",
  "agents": [
    { "id": "my-agent", "name": "My Agent", "status": "running" }
  ]
}
```

## Agent Endpoints

All agent endpoints are proxied through the orchestrator and gated by the agent's `expose[]` config. If an endpoint is not in `expose[]`, it returns `403 Forbidden`.

Base path: `http://localhost:4000/api/agents/{agent-id}/`

### GET /api/agents/{id}/status

Agent status, current task, memory files, last activity.

Requires: `expose: status` + JWT.

```bash
curl http://localhost:4000/api/agents/my-agent/status \
  -H "Authorization: Bearer {token}"
```

```json
{
  "agentId": "my-agent",
  "status": "running",
  "currentTask": null,
  "memoryFiles": ["report.md", "state.md"],
  "lastActivity": "2024-01-15T10:30:00Z",
  "uptime": 3600.5
}
```

### GET /api/agents/{id}/logs

Recent log entries from the agent's ring buffer (last 100 by default).

Requires: `expose: logs` + JWT.

```bash
curl "http://localhost:4000/api/agents/my-agent/logs?limit=50&level=info" \
  -H "Authorization: Bearer {token}"
```

Query params:
- `limit` — number of entries (default: 100)
- `level` — filter by level: `debug`, `info`, `warning`, `error`

```json
{
  "logs": [
    {
      "timestamp": "2024-01-15T10:30:00Z",
      "level": "info",
      "event": "task_completed",
      "taskId": "abc-123"
    }
  ]
}
```

### GET /api/agents/{id}/memory

List all files in the agent's memory.

Requires: `expose: memory` + JWT.

```bash
curl http://localhost:4000/api/agents/my-agent/memory \
  -H "Authorization: Bearer {token}"
```

```json
{
  "files": [
    {
      "filename": "report.md",
      "size": 4200,
      "lastModified": "2024-01-15T10:30:00Z",
      "lastCommitHash": "a1b2c3d4"
    }
  ]
}
```

### GET /api/agents/{id}/memory/{filename}

Get the content of a specific memory file.

Requires: `expose: memory` + JWT.

```bash
curl http://localhost:4000/api/agents/my-agent/memory/report.md \
  -H "Authorization: Bearer {token}"
```

```json
{
  "filename": "report.md",
  "content": "# Report\n\nFindings...",
  "lastModified": "2024-01-15T10:30:00Z",
  "lastCommitHash": "a1b2c3d4"
}
```

### PUT /api/agents/{id}/memory/{filename}

Write content to a memory file. Triggers git commit and RAG re-index.

Requires: `expose: memory` + JWT.

```bash
curl -X PUT http://localhost:4000/api/agents/my-agent/memory/notes.md \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{"content": "# Notes\n\nSome notes here."}'
```

```json
{ "ok": true, "filename": "notes.md" }
```

### GET /api/agents/{id}/tasks

List recent tasks for the agent.

Requires: `expose: tasks` + JWT.

```bash
curl "http://localhost:4000/api/agents/my-agent/tasks?limit=20" \
  -H "Authorization: Bearer {token}"
```

```json
{
  "tasks": [
    {
      "taskId": "abc-123",
      "status": "completed",
      "actionName": "analyse_topic",
      "senderId": "orchestrator",
      "createdAt": "2024-01-15T10:00:00Z",
      "completedAt": "2024-01-15T10:05:00Z"
    }
  ]
}
```

### POST /api/agents/{id}/chat

Send a message to the agent and get a task ID back. The agent processes it asynchronously using its full agentic loop (LLM + tools).

Requires: `expose: chat` + JWT.

```bash
curl -X POST http://localhost:4000/api/agents/my-agent/chat \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{"message": "Summarise the latest findings"}'
```

```json
{
  "taskId": "abc-123",
  "status": "processing"
}
```

Poll `/api/agents/{id}/tasks` or listen on WebSocket for completion.

### GET /api/agents/{id}/rag/status

RAG (Retrieval-Augmented Generation) status for the agent.

Requires: `expose: rag` + JWT.

```bash
curl http://localhost:4000/api/agents/my-agent/rag/status \
  -H "Authorization: Bearer {token}"
```

```json
{
  "enabled": true,
  "chunk_count": 142,
  "folders": ["/memory", "/storage/received"],
  "embedding_model": "all-MiniLM-L6-v2"
}
```

### POST /api/agents/{id}/rag/reindex

Force a full re-index of all RAG folders. Useful after bulk memory changes.

Requires: `expose: rag` + JWT.

```bash
curl -X POST http://localhost:4000/api/agents/my-agent/rag/reindex \
  -H "Authorization: Bearer {token}"
```

```json
{ "chunks_indexed": 142 }
```

### POST /api/agents/{id}/reload

Hot-reload the agent's config (re-reads `configs/agents/{id}.yaml` and restarts only that container). Requires JWT.

```bash
curl -X POST http://localhost:4000/api/agents/my-agent/reload \
  -H "Authorization: Bearer {token}"
```

```json
{ "ok": true, "agentId": "my-agent" }
```

## Webhooks

Webhooks are **public** — no JWT required. They trigger the agent's webhook-triggered actions.

### POST /webhooks/{agent-id}

Trigger an agent via webhook.

```bash
curl -X POST http://localhost:4000/webhooks/my-agent \
  -H "Content-Type: application/json" \
  -d '{
    "instruction": "Analyse the attached data",
    "payload": { "source": "external-system", "data": "..." }
  }'
```

```json
{ "ok": true, "taskId": "abc-123" }
```

The agent must have `type: webhook` in its `triggers` config.

## WebSocket Events

Connect to receive real-time events from all agents.

```
WS ws://localhost:4000/ws?token={jwt}
```

```javascript
const ws = new WebSocket(`ws://localhost:4000/ws?token=${token}`);
ws.onmessage = (e) => {
  const event = JSON.parse(e.data);
  console.log(event.type, event);
};
```

### Event Types

```typescript
// Agent changed status (starting, running, stopped)
{ type: "agent:status"; agentId: string; systemId: string; status: string; timestamp: string }

// Agent emitted a log line
{ type: "agent:log"; agentId: string; systemId: string; level: string; message: string; timestamp: string }

// Agent wrote a file to memory
{ type: "agent:memory:updated"; agentId: string; systemId: string; file: string; commitHash: string; timestamp: string }

// Agent started processing a task
{ type: "agent:task:started"; agentId: string; systemId: string; taskId: string; timestamp: string }

// Agent completed a task
{ type: "agent:task:completed"; agentId: string; systemId: string; taskId: string; output: string; timestamp: string }

// Agent failed a task
{ type: "agent:task:failed"; agentId: string; systemId: string; taskId: string; error: string; timestamp: string }

// System-wide status change
{ type: "system:status"; systemId: string; status: string; timestamp: string }
```

## Expose Configuration

Each agent's `expose[]` list in its YAML config controls which endpoints are accessible through the orchestrator proxy. Endpoints not in `expose[]` return `403 Forbidden`.

```yaml
expose:
  - status    # GET /api/agents/{id}/status
  - logs      # GET /api/agents/{id}/logs
  - memory    # GET/PUT /api/agents/{id}/memory/*
  - tasks     # GET /api/agents/{id}/tasks
  - chat      # POST /api/agents/{id}/chat
  - rag       # GET /api/agents/{id}/rag/status, POST /api/agents/{id}/rag/reindex
```

To change what's exposed: edit `configs/agents/{id}.yaml`, then call `POST /api/agents/{id}/reload`.

## Error Responses

All errors follow this format:

```json
{ "error": "Human-readable message", "code": "ERROR_CODE" }
```

| HTTP Status | Code | Meaning |
|---|---|---|
| 401 | `UNAUTHORIZED` | Missing or invalid JWT |
| 403 | `FORBIDDEN` | Endpoint not in agent's `expose[]` |
| 404 | `NOT_FOUND` | Agent ID not found |
| 500 | `INTERNAL_ERROR` | Unexpected server error |

## Quick Reference

```bash
# 1. Get a token
TOKEN=$(curl -s -X POST http://localhost:4000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"password":"your-api-password"}' | jq -r .token)

# 2. Check system health
curl http://localhost:4000/health

# 3. System status
curl http://localhost:4000/api/system/status -H "Authorization: Bearer $TOKEN"

# 4. Agent status
curl http://localhost:4000/api/agents/{id}/status -H "Authorization: Bearer $TOKEN"

# 5. Read a memory file
curl http://localhost:4000/api/agents/{id}/memory/report.md -H "Authorization: Bearer $TOKEN"

# 6. Trigger via webhook (no token needed)
curl -X POST http://localhost:4000/webhooks/{agent-id} \
  -H "Content-Type: application/json" \
  -d '{"instruction": "Do something"}'

# 7. Chat with an agent
curl -X POST http://localhost:4000/api/agents/{id}/chat \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"message": "What did you find?"}'

# 8. Hot-reload agent config after editing YAML
curl -X POST http://localhost:4000/api/agents/{id}/reload -H "Authorization: Bearer $TOKEN"
```
