Setting up a multi-agent AI system that actually works in a production environment is a massive engineering headache.

Usually, you have to spend weeks dealing with:
- Writing custom API routers and schema mapping for agent-to-agent communication
- Setting up message queues, Redis layers, and database volumes to manage state
- Handling file-based handoffs, race conditions, and vector search index synchronization
- Manually writing Docker Compose configurations and mounting files for each service

What if you could visually drag and drop your agents, connect them together, and run the entire stack in one click?

I built AgentDock to solve this.

AgentDock is a visual design-to-code platform for multi-agent systems. It splits the workflow into two independent parts: a visual Builder UI to design and verify systems, and a self-contained containerized Runtime to execute them.

THE WORKFLOW:
1. DESIGN (Builder UI): Layout agent nodes visually on a drag-and-drop canvas, or describe your system in plain English to auto-generate the workflow. Configure system prompts, temperatures, and tools directly.
2. VERIFY: The backend runs programmatic and LLM-based verification loops to validate configurations, automatically repairing broken triggers or inputs.
3. EXPORT & DEPLOY: Export the system into a self-contained ZIP file with standard Dockerfiles and readable YAML configs—no proprietary lock-in. Unzip it on any machine and run "docker compose up -d" to spin up the orchestrator, LLM gateway, Redis queue, persistent memory volumes, and agents.

It also automatically builds and serves a dedicated RUNTIME CONSOLE on port 5173. The console is fully dynamic—it queries the active workflow topology to build a live node graph, letting you trace agent execution statuses, read/download file handoffs, and stream stdout/stderr container logs in real-time over WebSockets.

UNDER THE HOOD, THE STACK BUNDLES:
- 52 pre-built Model Context Protocol (MCP) integrations (including Postgres, Gmail, Notion, Slack, and web search)
- A self-learning memory loop using ChromaDB that saves successful agent interactions back to persistent volume storage
- First-class support for running local LLMs (like Llama 3 or Qwen) out-of-the-box via Ollama, or external APIs like OpenAI, Gemini, and Anthropic
- A clean, modern tech stack built with Bun, TypeScript, Vite, Tailwind CSS v4, and FastAPI/Python
