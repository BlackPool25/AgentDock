# AgentDock Runtime Console

A premium, high-fidelity React dashboard built with **TypeScript**, **Tailwind CSS v4**, and **Vite**, designed to manage, monitor, and coordinate any generated multi-agent **AgentDock** pipeline. 

This console connects directly to the Hono/TypeScript orchestrator backend (`http://localhost:4000`) and provides full visibility and control over agent statuses, tasks, configurations, semantic memories, and live execution streams.

## 🏗️ System Architecture & Connectivity

```mermaid
graph TD
  A[React Frontend Console] -->|POST /auth/login| B(Orchestrator Backend)
  A -->|ws://localhost:4000/ws| B
  A -->|POST /webhooks/{entrypoint-agent}| B
  A -->|GET/PUT /api/agents/...| B
  B -->|Docker Internal Proxy| C[Agent 1]
  B -->|Docker Internal Proxy| D[Agent 2]
  B -->|Docker Internal Proxy| E[Agent 3]
```

### 1. Security & Authentication
Authentication is streamlined via a single-password gate configured in the orchestrator's `.env` file. On successful verification, a JWT token is stored securely in `localStorage` and sent with all API requests as a `Bearer` token inside the `Authorization` header.

### 2. Dynamic Topology Discovery
On mount, the console automatically queries `/topology` to construct the node map and connections from the active `workflow.yaml`. It dynamically locates coordinates, agent configuration payloads, and connection paths without hardcoding, making it fully generalizable to any user-designed agent architecture.

### 3. Entrypoint Webhook Dispatch
When an instruction is submitted through the chatbox, the system:
1.  **Auto-detects the pipeline entrypoint**: It finds the root agent node that has no incoming edges.
2.  **Seeds default profile**: It executes a `PUT` request to `/api/agents/{entrypoint}/memory/profiles/admin.md` to guarantee a default profile is present for the LLM to inspect.
3.  **Triggers webhook**: It fires a `POST` request to the orchestrator webhook at `/webhooks/{entrypoint}` with the instruction and context.

### 4. Live Log WebSocket Hub
Connects to `ws://localhost:4000/ws?token={token}` to catch all logs in real-time. Logs are buffered on the frontend and are fully filterable by log level and specific agent identifiers.

## 🚦 Getting Started & Local Run

### 1. Run inside Standalone System
The runtime console is included by default in all exports generated by AgentDock. Simply unzip your generated project and execute:
```bash
docker compose up -d
```
The console will be built and served at `http://localhost:5173`.

### 2. Local Development Run
To run the console developer server with live reload:
```bash
# From within apps/runtime-console
bun install
bun run dev
```

## 🛠️ Future Rework & Extension Guidelines

1.  **Adjusting Connections & Layout**:
    *   All coordinates and connections are loaded directly from the orchestrator. To change the visual canvas layout, modify the `configs/workflow.yaml` file in the orchestrator configuration directory.
2.  **Custom Webhooks**:
    *   The webhook route uses an API key that is identical to the target `agentId` for simple dev triggers. If you add secondary webhooks, they can be fired via `POST /webhooks/{agentId}`.
